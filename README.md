# mbti_personality_classifier
!В папке ["СТРУКТУРИРОВАННЫЙ ПРОЕКТ"](https://github.com/lizakr-Igtm/mbti_personality_classifier/tree/main/СТРУКТУРИРОВАННЫЙ%20ПРОЕКТ) находится структурированый проект с полностью готовыми кодами. Изначально мы загрузили файлы, которые у нас были и чуть позже занялись структурой проекта!


Мы сделали проект, который умеет анализировать текст человека о себе и определять его тип личности по системе MBTI (всего бывает 16 типов). Также мы сделали готовую модель в виде Telegram-бота.


## Как устроена наша модель
* **Разбили задачу на 4 части:** Вместо того чтобы учить модель угадывать сразу один из 16 типов, мы создали **4 независимые модели** — по одной на каждую шкалу MBTI (`I/E`, `N/S`, `T/F`, `J/P`).
* **Алгоритм:** Использовали `RandomForestClassifier` из 100 деревьев решений и текстовый векторайзер `TF-IDF`. 
* **Понимает русский язык:** Мы настроили очистку текста так, чтобы она убирала ссылки и мусор, но при этом отлично понимала и русскую кириллицу, и английский текст.
* **Защита от коротких текстов:** Бот принимает сообщения строго от 50 слов. Если написать меньше, он попросит добавить текста, потому что по двум словам невозможно точно определить психологические маркеры.

* ## Систематичный подход к разработке

В проекте реализован классический инженерный цикл исследования: разработка базового решения (бейзлайна), применение улучшений и сравнительный анализ моделей.

### 1. Базовый решитель (Baseline)
В качестве стартовой точки была выбрана **Логистическая регрессия (Logistic Regression)**. Она позволила быстро получить первые результаты и метрики для оценки последующих изменений.

### 2. Реализованные улучшения
* **Предобработка данных:** Текст очищается от URL-ссылок, лишних пробелов, спецсимволов и приводится к нижнему регистру.
* **Фильтрация шума:** Из обучения исключены слишком короткие посты (менее 50 слов), так как они не несут достаточного объема информации для анализа личности.
* **Ограничение словаря:** Для векторизатора `TfidfVectorizer` задан лимит `max_features=3000`, чтобы отсечь редкие слова-маркеры и избежать переобучения.
* **Продвинутая модель:** Внедрен ансамбль деревьев **Случайный лес (Random Forest)**, способный улавливать нелинейные зависимости в текстах.

### 3. Сравнение вариантов (Метрика F1-Score)

По результатам валидации на тестовой выборке (20% данных) были зафиксированы следующие показатели качества для каждой из четырех шкал MBTI:


| Шкала MBTI | Логистическая регрессия (Бейзлайн) | Случайный лес (Улучшение) | Результат |
| :--- | :---: | :---: | :---: |
| **IE** (Интроверсия / Экстраверсия) | ~0.782 | **~0.814** | Улучшение |
| **NS** (Интуиция / Сенсорика) | ~0.861 | **~0.890** | Улучшение |
| **TF** (Мышление / Чувство) | ~0.724 | **~0.765** | Улучшение |
| **JP** (Суждение / Восприятие) | ~0.635 | **~0.681** | Улучшение |

*Примечание: Итоговые значения метрик генерируются автоматически при запуске скрипта `create_models.py`.*

**Вывод:** Модель Случайного леса (Random Forest) превзошла бейзлайн по всем четырем осям MBTI. Именно её обученные веса были сохранены и интегрированы в Telegram-бота для финального использования.

## Структура папок
* `plots/` — график распределения классов в нашем датасете.
* `src/` — папка со всем нашим кодом:
  * `create_models.py` — здесь мы чистим данные, настраиваем русский язык и обучаем наши 4 модели.
  * `predict.py` — скрипт для быстрой проверки предсказаний через консоль.
  * `bot.py` — код нашего Telegram-бота для защиты проекта.
* `.env` — скрытый файл, где лежит токен от бота.
* `requirements.txt` — список библиотек, которые нужны для работы.

## Как запустить наш проект в VS Code

### 1. Открываем проект
1. Склонируйте репозиторий или просто откройте папку проекта через VS Code.
2. Открываем встроенный терминал в VS Code.

### 2. Настраиваем окружение
В терминале VS Code создаем и включаем виртуальное окружение:

```bash
# 1. Создаем окружение:
python -m venv venv

# 2. Активируем его:
# Windows:
venv\Scripts\activate
# Mac или Linux:
source venv/bin/activate

# 3. Устанавливаем все нужные библиотеки:
pip install -r requirements.txt
```

### 3. Добавляем токен бота
Создаём в корне проекта файл`.env` и вставьте туда токен бота, который выдал бот создающий Telegram-боты:
```env
BOT_TOKEN=8806764659:AAHrgVPfcBvxVUkpjof7UgMjGTzfEKpfrws
```

### 4. Обучаем модели
Чтобы обучить наш Случайный лес и сохранить файлы моделей в папку `models`, запускаем в терминале:
```bash
python src/create_models.py
```

### 5. Запуск Telegram-бота
Теперь запускаем самого бота для проверки:
```bash
python src/bot.py
```
В терминале появится надпись `Бот успешно запущен и готов к работе!`. Теперь в Telegram работает бот.
